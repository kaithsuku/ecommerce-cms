## Design Context

### Users
Small-store admins and operations staff use this CMS while updating product content,
stock, categories, and order status. They need speed, clarity, and confidence more
than decorative ecommerce marketing.

### Brand Personality
Premium, sharp, calm. The interface should feel editorial and operational: like a
well-designed commerce back office, not a generic analytics dashboard.

### Aesthetic Direction
Bold-minimal editorial admin UI. Use light surfaces, strong typographic scale,
restrained color, clear tables, and deliberate spacing. Avoid purple-blue
gradients, glassmorphism, neon dark mode, nested cards, and generic AI dashboard
visuals.

### Design Principles
- Make product and order operations scannable before they are decorative.
- Use contrast, type scale, and spacing as the primary visual tools.
- Keep containers sparse; use tables and forms with strong hierarchy.
- Make every visible control directly useful to the admin workflow.
- Preserve accessibility with clear focus states, readable contrast, and reduced motion support.
